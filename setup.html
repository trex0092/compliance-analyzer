🛡 HAWKEYE STERLING — Setup Wizard

Browser-only setup. No terminal. No command line. Zero developer intervention.

Step 1 — Accounts you need (open in new tabs)

Create each account in its own tab. Come back here when you have all four values.

  1. Anthropic Console → API Keys — click Create Key, label it hawkeye, copy the sk-ant-… value
  2. Netlify — sign up (free tier is enough)
  3. Asana Developer Console — click + New access token, copy the 1/… value
  4. In Asana, open your workspace home. The URL contains the workspace ID (the long number after /0/). Copy that too.
Save the four values somewhere. You'll paste them into Step 4 below.

Step 2 — Fork the repo on GitHub

This is a pure click-through, no terminal.

  1. Open github.com/trex0092/compliance-analyzer
  2. Click the Fork button (top right)
  3. Accept the defaults → click Create fork
  4. Wait ~10 seconds for GitHub to copy the repo into your account
  5. Leave the forked repo tab open — you'll use it in Step 3

Step 3 — Connect Netlify to your GitHub fork

  1. Open Netlify dashboard
  2. Click Add new site → Import an existing project
  3. Click Deploy with GitHub → authorise Netlify to access your GitHub
  4. Pick the compliance-analyzer fork from the list
  5. Leave every default → click Deploy site
  6. Wait 3-5 minutes. The first deploy will succeed but some endpoints return 503 because env vars aren't set yet — that's expected and fine.
  7. When the deploy finishes, copy your site URL (something like https://dreamy-banana-123abc.netlify.app)
After the first deploy, the tool will warn about missing env vars. Ignore the warnings for now — Step 5 fixes them.

Step 4 — Generate your secrets

Click the button. Your browser will generate cryptographically secure random values using crypto.getRandomValues() — the same primitive Netlify's own secret generator uses. Nothing is sent to the network.

Step 5 — Paste these into Netlify env vars

Click Copy below, then open Netlify → your site → Site configuration → Environment variables → Import from a .env file → paste.

(click "Generate secrets" above to see your env var block)

After you paste + save in Netlify, Netlify will automatically redeploy. Wait ~2 minutes.

Step 6 — Verify the tool is alive

Once Netlify finishes the redeploy, click the button below to verify every endpoint is responding.

(click verify to check)

Step 7 — Upload your first customer cohort

Paste your CSV in the exact schema below. Rows with errors are reported per-row; the whole import does not fail.

id,name,tenantId,aliases,dateOfBirth,nationality,lastScreenedAtIso
cust-1,Acme Trading LLC,tenant-a,ACME;Acme Trading,,AE,
cust-2,Bob Smith,tenant-a,Robert Smith,1985-04-15,GB,2026-04-10T00:00:00Z
(paste a CSV and click upload)

Step 8 — Bootstrap Asana for this tenant

This creates the Asana project, the 14 standard sections, the 15 workspace custom fields, and the webhook — all in one click. Idempotent: safe to re-run.

(click to provision)

Step 9 — Provision KYC / CDD Tracker sections

One click to provision the tenant-agnostic "KYC / CDD TRACKER — ALL ENTITIES" Asana project with all 16 canonical workflow sections (intake, exception lanes, CDD, EDD, approvals, terminal). Idempotent: existing sections are reused by name, the empty "Untitled section" placeholder is removed, and operator custom sections are preserved untouched.

(enter project GID and click to provision)

Step 10 — Scan for lumped entity tasks

Compliance rule: each legal entity must have its own dedicated Asana task. Lumping multiple entities (e.g. "GRAMALTIN AS / NAPLES LLC / ZOE FZE — review") into one task breaks the per-entity audit trail required by FDL Art.24 and Cabinet Res 134/2025 Art.7-10. This scanner is read-only: it lists every task in the project whose title mentions 2+ entities, so you can split them manually in Asana.

(enter project GID and click to scan)

Step 11 — Run TM scan (Transaction Monitoring)

Runs the Transaction Monitoring Brain across all customer transactions in the blob store. The brain applies 6 threshold rules (FDL Art.15-16, MoE Circular 08/AML/2021), 6 FATF typology patterns (smurfing, layering, round-trip, TBML, hawala, shell-passthrough), and 4 statistical detectors (Benford, Z-score, velocity burst, dormancy break). Default is dry-run; tick "Dispatch" to create Asana tasks for flagged customers.

(click to run transaction monitoring scan)

Step 12 — You are live 🎉

When Steps 6, 7, 8, 9, 10 and 11 all show OK, open the tool in a new tab and start using it.

Step 13 — Post-setup checklist (human work, not the tool)

  • Register your firm with the UAE FIU goAML portal
  • Sign the Anthropic Data Processing Agreement (required for any EU-resident customer)
  • Train your MLRO + CO + Board — run a drill in demo mode first
  • Publish your AML policy via the policy editor (3-signature flow: MLRO → CO → Board)
  • Set the CO's email / Slack webhook for alerts (HAWKEYE_ALERT_EMAIL / HAWKEYE_ALERT_SLACK_URL in Netlify env vars)
