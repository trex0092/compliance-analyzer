# Screening Report — Standard Template + Samples

**Purpose.** Single canonical format for every subject screened through
Hawkeye. Produced immediately after `/screen` completes, stored in the
audit chain under FDL No.10/2025 Art.24 (10-year retention), and
available for export to PDF for MoE / LBMA / internal audit.

**Verdict taxonomy.** Every report MUST resolve to exactly one of:

| Verdict         | Meaning                                                                  | Regulatory trigger                       |
|-----------------|--------------------------------------------------------------------------|------------------------------------------|
| NEGATIVE        | No match on any list; no adverse media; no PEP signal.                   | Proceed with standard CDD.               |
| POSITIVE        | Confirmed match (score >= 0.90). Freeze + file.                          | Cabinet Res 74/2020 Art.4-7; FDL Art.35. |
| PARTIAL MATCH   | Potential match (0.50-0.89). Four-eyes review required before disposition. | FDL Art.20-21; Cabinet Res 134/2025 Art.14. |
| FALSE POSITIVE  | Match initially flagged, independently reviewed, dismissed with evidence. | FDL Art.20-21 (MLRO discretion + audit). |

**Standard sections** (every report, no exceptions):

1. Report header (ID, classification, generated by, timestamp)
2. Subject identity
3. Screening parameters (lists, providers, duration)
4. Screening outcome (verdict, confidence, highest match)
5. Findings (sanctions, PEP, adverse media, UBO / layering / shell)
6. Risk assessment (rating, CDD level, context multipliers)
7. MLRO disposition (action, rationale, four-eyes, filing obligations)
8. Regulatory filings + deadlines
9. Audit trail + retention

## Usage Notes (for the MLRO)

- **Do not hand-write these reports.** The `/screen` skill produces the
  live JSON; the exporter renders it into this exact template. Hand-
  writing loses the tamper-evident seal and breaks Art.24 retention.
- **Never include raw API keys, tokens, or internal passwords** in a
  screening report, even in the audit trail — use blob references.
- **Every POSITIVE report must carry the no-tipping-off banner** at the
  top. Distribution must be limited to MLRO, CO, SMT approver, auditors.
- **PARTIAL MATCH is a state, not a disposition.** Every PARTIAL MATCH
  must resolve within 24h into either POSITIVE (freeze workflow) or
  FALSE POSITIVE (documented dismissal). Open PARTIAL MATCH past 24h =
  Art.20-21 escalation to CO + SMT.
- **FALSE POSITIVE is not an absence of risk.** A dismissal-with-evidence
  report must cite the specific disqualifying data points (DOB, POB,
  document trail); "looks different" is not acceptable.

## Regulatory Citations (all report verdicts)

- FDL No.10/2025 Art.20-21 (CO / MLRO duties, escalation)
- FDL No.10/2025 Art.24 (10-year audit retention)
- FDL No.10/2025 Art.26-27 (STR filing via goAML)
- FDL No.10/2025 Art.29 (no tipping off — applies to all verdicts involving sanctions, PEP, or STR triggers)
- FDL No.10/2025 Art.35 (list coverage — UN, OFAC, EU, UK, UAE)
- Cabinet Res 74/2020 Art.4-7 (asset freeze within 24h, EOCN + CNMR)
- Cabinet Res 134/2025 Art.7 (CDD tiers: SDD / CDD / EDD)
- Cabinet Res 134/2025 Art.14 (EDD + Senior Management approval)
- Cabinet Decision 109/2023 (UBO 25% threshold, 15-working-day re-verification)
- Cabinet Res 156/2025 (PF + dual-use screening — always applied on Russia/Iran/DPRK-linked subjects)
